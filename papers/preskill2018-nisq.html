So why is it taking so long? What is it about quantumterm·key computing that’s so difficult? The history of classicalterm·key computing teaches us that when hardware becomes available that stimulates and accelerates the development of new algorithms. 6.5 Quantumterm·key deep learning

Machine learning is transforming technology and having a big impact on science as well, so it is natural to wonder about the potential of combining machine learning with quantumterm·key technology. There are a variety of different notions of “quantumterm·key machine learning.” Much of the literature on the subject builds on quantumterm·key algorithms that speed up linear algebra and related tasks [37, 38], and I’ll address such applications in the ensuing subsections.

First, though, I’ll comment on the potential of quantumterm·key deep learning [39]. Deep learn- ing itself has become a broad subject [27], but to be concrete let us contemplate a restricted Boltzmann machine. This may be regarded as a spin system in thermal equilibrium at a low but nonzero temperature, with many hidden layers of spins separating the input and output. (The word “restricted” means that there are no couplings among spins within a given layer; instead only spins in successive layers are coupled.) The system may have millions of coupling parameters, which are optimized during a training phase to achieve a desired joint probability distribution for input and output. The training may be super- vised, for example the network might be taught to recognize a labeled set of photos, or unsupervised, in which case the goal is to identify patterns in an unlabeled training set of data. The quantumterm·key analog of such a machine could have a similar structure and function, but where the spins are qubitsterm·key rather than classicalterm·key bits. It may be that quantumterm·key deep learning networks have advantages over classicalterm·key ones; for example they might be easier to train for some purposes. But we don’t really know — it’s another opportunity to try it and see how well it works.

One possible reason for being hopeful about the potential of quantumterm·key machine learning rests on the concept known as QRAM — quantumterm·key random access memory. By QRAM we mean representing a large amount of classicalterm·key data very succinctly, by encoding a vector with N components in just log N qubitsterm·key. But typical proposals for quantumterm·key machine learning applications are confounded by severe input/output bottlenecks. For applications to large classicalterm·key data sets one should take into account the cost of encoding the input into QRAM, which can nullify the potential advantages. Furthermore, the output from a quantumterm·key network is itself a quantumterm·key state; to get useful information we need to measure the state, and we can acquire no more than log N bits of classicalterm·key information in a single- shot measurement of log N qubitsterm·key.

These bottlenecks arise when we try to train a quantumterm·key network to learn about cor- relations in classicalterm·key data. Perhaps it is more natural to think about quantumterm·key machine learning in a setting where both the input and the output are quantumterm·key states. Hence we might expect a quantumterm·key network to have advantages over a classicalterm·key one when used, for example, to control a complex quantumterm·key system. More broadly, it seems plausible that quantumterm·key deep learning would be more powerful than its classicalterm·key counterpart when trying to learn probability distributions for which quantumterm·key entanglementterm·dom has a significant role. That is, quantumterm·key deep learning networks might be very well suited for quantumterm·key tasks, but for applications of deep learning that are widely pursued today it is unclear why quantumterm·key networks would have an advantage.

On the encouraging side, it’s worth noting that a quantumterm·key deep learning machine won’t necessarily need to be a general purpose circuit-based quantumterm·key computer. It might be a special purpose device — perhaps a quantumterm·key annealer, if not too noisy.

6.6 Quantumterm·key matrix inversion

QRAM has further implications for quantumterm·key algorithms. In particular, the task of matrix inversion admits an exponential quantumterm·key speedup, which could have many applications.

The algorithm we call HHL [40] (after the three authors who first discussed it nine years ago) takes as input a succinct representation of a very large N ×N matrix A, which has to be sufficiently sparse and well conditioned, and also as input an N -component vector |bi, encoded in QRAM as a quantumterm·key state of log N qubitsterm·key. The output of the quantumterm·key algorithm is (with a small error) the quantumterm·key state |A−1bi, the result of applying the inverse of the matrix A to the input vector |bi. This quantumterm·key algorithm runs in time O(log N ), an exponential speedup relative to classicalterm·key matrix inversion (for fixed error and for fixed sparsity and condition number of A).

However, aside from the caveat that the matrix A must be sparse and well conditioned, note that both the input vector |bi and the output vector |A−1bi are quantumterm·key states. Once we have the output state, we can perform measurements to learn features of the output vector A−1b, and by repeating the scheme many times we can learn about the output vector in greater detail. But if we are trying to apply this matrix inversion algorithm to classicalterm·key input data, we need to take into account the potential cost of loading that classicalterm·key data into QRAM, which could nullify the exponential speedup. Alternatively, loading classicalterm·key data into QRAM might be avoided by computing |bi on a quantumterm·key computer rather than entering it from a database.

We do have good reason to believe this quantumterm·key matrix inversion algorithm is powerful, because it solves what we call a BQP-complete problem. That is, any problem that can be solved efficiently with a quantumterm·key computer can be encoded as an instance of this matrix inversion problem. And a number of potentially useful applications have been suggested. For example, we can use this matrix inversion algorithm to find approximate solutions to classicalterm·key linear field equations [41, 42, 43] (after discretizing the field on a spatial lattice, and applying a suitable preconditioner). This procedure could be used, say, to solve the equations of electromagnetism in a complex three-dimensional geometry, for the purpose of optimizing the performance of an antenna.

I expect HHL to have high-impact applications eventually, but it is not likely to be fea- sible in the NISQterm·key era. The algorithm may just be too expensive to be executed successfully by a quantumterm·key computer which doesn’t use error correction.

6.7 Quantumterm·key recommendation systems

Another recently proposed quantumterm·key algorithm [44] provides an exponential speedup over the best currently known classicalterm·key algorithm for a useful task: making high-value recom- mendations. The goal is to recommend a product to a customer that the customer will probably like, based on limited knowledge of the preferences of that customer and other customers.

To formulate a simplified version of the problem, consider m customers and n products, and let P be a binary m×n preference matrix such that Pai = 1 if customer a likes product i and Pai = 0 if customer a dislikes product i. 6.8 Quantumterm·key semidefinite programming

Semidefinite programming is the task of optimizing a linear function subject to matrix inequality constraints. Specifically, given as input the m+1 N ×N Hermitian matrices {C, A1, A2, . . . , Am} and real numbers {b1, b2, . . . , bm}, the problem is to find a posi- tive semidefinite N × N matrix X that maximizes tr (CX) subject to the m constraints tr (AiX) ≤ bi. Convex optimization problems of this type have widespread applications, and can be solved classically in poly(m, N ) time.

A recently discovered quantumterm·key algorithm [46, 47] finds an approximate solution to the problem in time polylog(N ), an exponential speedup. In the quantumterm·key algorithm, the output is a quantumterm·key state, a density operator ρ which approximates the optimal matrix X. This quantumterm·key state can then be measured to extract information about X, and the procedure can be repeated many times to learn about X is more detail.

However, in the analysis of the quantumterm·key algorithm one assumes the availability of an initial quantumterm·key state of a particular form, the thermal Gibbs state associated with a Hamiltonian which is a linear combination of the input matrices for the semidefinite program. Whether the exponential speedup can actually be realized, then, depends on whether this Gibbs state can be prepared efficiently using a quantumterm·key computer.

For semidefinite programs with the right structure, the efficient preparation of the Gibbs state is possible, and therefore the exponential speedup really is feasible. In par- ticular, the quantumterm·key algorithm achieves an exponential speedup if all the input matrices for the semidefinite program have low rank, or if the corresponding Hamiltonian system thermalizes rapidly. It is not yet clear to what extent these properties apply to semidef- inite programs of practical interest; clarifying that issue is an important goal for further research.

It is intriguing that the crucial step in the quantumterm·key algorithm is the preparation of a thermal state at a nonzero temperature. 6.9 Quantumterm·key simulation

As I have already emphasized, we expect quantumterm·key computersterm·key to be very well suited for studying the properties of highly entangled systems of many particles. 6.10 Digital vs. analog quantumterm·key simulation

Having emphasized the potential value of simulating quantumterm·key dynamics, I should comment about the distinction between analog and digital quantumterm·key simulation. When we speak of an analog quantumterm·key simulator we mean a system with many qubitsterm·key whose dynamics resembles the dynamics of a model system we are trying to study and understand. In contrast, a digital quantumterm·key simulator is a gate-based universal quantumterm·key computer which can be used to simulate any physical system of interest when suitably programmed, and can also be used for other purposes.

Analog quantumterm·key simulation has been a very vibrant area of research for the past 15 years, while digital quantumterm·key simulation with general purpose circuit-based quantumterm·key com- puters is just now getting started. But some of the same experimental platforms, for example trapped ions and superconducting circuits, can be used for both purposes. Ana- log quantumterm·key simulators have been getting notably more sophisticated, and are already being employed to study quantumterm·key dynamics in regimes which may be beyond the reach of classicalterm·key simulators [51, 52]. I’ve already emphasized repeatedly that it will probably be a long time before we have fault-tolerantterm·key quantumterm·key computersterm·key solving hard problems. Nevertheless, we can make sig- nificant progress in the near term by developing better methods and hardware for im- plementing quantum error correctionterm·mth, guided by relatively small-scale experiments with quantumterm·key error-correcting codes. Now is a privileged time in the history of science and technology, as we are witnessing the opening of the NISQterm·key era (where NISQterm·key = noisy intermediate-scale quantumterm·key). We’ll soon have the opportunity to experiment with NISQterm·key technology to see what it can do.
