1.2 Quantumterm·key Mechanical Algorithms A good starting point to think of quantumterm·key mechanical algo- rithms is probabilistic algorithms [BV93] (e.g. simu- lated annealing). In these algorithms, instead of having the system in a specified state, it is in a distribution over various statesterm·key with a certain probabilityterm·key of being in each state. At each step, there is a certain probabilityterm·key of mak- ing a transition from one state to another. The evolution of the system is obtained by premultiplying this proba- bility vector (that describes the distribution of probabili- ties over various statesterm·key) by a state transition matrix. Knowing the initial distribution and the state transition matrix, it is possible in principle to calculate the distri- bution at any instant in time.

Just like classical probabilistic algorithms, quantumterm·key mechanical algorithms work with a probabilityterm·key distribution over various statesterm·key. However, unlike classi- cal systems, the probabilityterm·key vector does not completely describe the system. In order to completely describe the system we need the amplitudeterm·key in each state which is a complex number. The evolution of the system is obtained by premultiplying this amplitudeterm·key vector (that describes the distribution of amplitudes over various statesterm·key) by a transition matrix, the entries of which are complex in general. The probabilities in any state are given by the square of the absolute values of the ampli- tude in that state. It can be shown that in order to con- serve probabilities, the state transition matrix has to be unitaryterm·key [BV93].

7. Implementation considerations This algorithmterm·key is likely to be simpler to implement as com- pared to other quantumterm·key mechanical algorithms for the following reasons:

(i) The only operations required are, first, the Walsh-Hadamard transform, and second, the conditional phaseterm·key shift operation both of which are relatively easy as compared to operations required for other quantumterm·key mechanical algorithms [BCDP96].

(ii) Quantumterm·key mechanical algorithms based on the Walsh-Hadamard transform are likely to be much sim- pler to implement than those based on the “large scale

Fourier transform”.

(iii) The conditional phaseterm·key shift would be much eas- ier to implement if the algorithmterm·key was used in the mode where the function at each point was computed rather than retrieved form memory. This would eliminate the storage requirements in quantumterm·key memory.

(iv) In case the elements had to be retrieved from a table (instead of being computed as discussed in (iii)), in principle it should be possible to store the data in classi- cal memory and only the sampling system need be quantumterm·key mechanical.
